## Findings

### CRITICAL — bugs blocking customer onboarding

#### F2 — `project_setup.py` first-run nepersistuje typed Status field_ids
**Reprodukce:** čistá instalace + `project_setup.py` na fresh GH Project.
**Dopad:** `field_ids` v `.edpa/config/edpa.yaml` obsahuje jen `Epic Status` a `Feature Status`,
chybí `Initiative Status` a `Story Status`. Pole na GH však existují (script je vytvoří),
ale jejich IDs/options se nezapíšou. Sync push/pull pro Initiative/Story status pak selže.
**Workaround:** spustit setup znovu nebo `sync.py setup-refresh` (po té všech 20 polí + 35 options OK).

#### F3 — `project_setup.py` NENÍ idempotentní
**Reprodukce:** druhý `project_setup.py` se stejným `--project-title`.
**Dopad:** vytvoří **nový Project #N+1** místo reuse stávajícího. Test plán §2.6 očekává
"Project might already exist — reusing #N" — script tuhle větev nemá.

#### F4 — Idempotence-fail vytváří duplicitní issues
**Reprodukce:** druhý setup → pro každý YAML item se vytvoří nové GitHub issue.
**Dopad:** v repo skončí duplicates `I-1, E-1, F-1, S-1` (issues #1-4 + #5-8), `issue_map.yaml`
přepsán na novou sadu — staré issues osiřelé.

#### F13 — `transitions.py` netracuje Story status changes
**Reprodukce:** úprava status v `.edpa/backlog/stories/*.yaml`, sync commit, run transitions.py.
**Dopad:** `TRACKED_DIRS = {features, epics, initiatives}` — Story changes jsou ignorovány.
Pro gates mode pak engine nevidí story-level transitions, krediture jen Story=Done snapshot
+ feature/epic/initiative gates. Mid-life kreditace na Story neexistuje, přestože dokumentace
i CW heuristics tabulka pracují s rolemi key/reviewer typicky vázanými na story-level práci.

#### F14 + F16 — Engine vrací 0 hodin při schema mismatch v `contributors[]`
**Reprodukce:** YAML stories s `contributors[].role: Dev/Arch/QA/PM` a `weight: 0.7`.
**Dopad:** engine `load_backlog_items` ignoruje neznámé role a unknown `weight` field —
nemapuje je na evidence signály. Žádný warning, žádná validation. Engine projde s
"All invariants passed: YES" + 0 derived hours + 0 items per person.
**Správné schéma (zjištěno čtením engine.py:407-433):**
```yaml
contributors:
  - person: bob-dev
    role: owner       # owner | key | reviewer | consulted (evidence role)
    cw: 0.7           # NE "weight"
```
**Doporučení:**
1. `validate_syntax.py` musí enforce schema (enum role, required cw)
2. engine warn at startup: "0 evidence pairs found in N stories — check schema"
3. Aktualizovat `templates/people.yaml.tmpl` + `docs/QUICKSTART.md` ukázkou správného schématu

#### F17 — `/edpa:reports` skill nemá batch CLI
**Reprodukce:** `python3 .claude/edpa/scripts/...` — žádný `reports.py` neexistuje.
**Dopad:** per-person `timesheet-<id>.md` lze vyrobit jen přes interaktivní Claude Code session
+ skill execution. Test plán §10.1 implikuje samostatný batch entry-point, který chybí.
Engine sám produkuje `summary.xlsx` a `item-costs.xlsx` + snapshot, ale ne MD timesheets.
Pro reálný PI close (kdy účetní/PM chtějí MD reporty) je to zbytečná závislost na LLM.

### HIGH — funkční, ale neudržuje slíbené chování

#### F8 — Iteration option missing po setup
`project_setup.py` vytvoří field `Iteration` jen s options `TBD`. Když backlog story má
`iteration: PI-2026-1.1`, sync push selže `[failed: no option_id for 'Iteration':'PI-2026-1.1']`.
Bootstrap by měl pre-vyplnit options ze všech `.edpa/iterations/*.yaml`.

#### F10 — `sync conflicts` nedetekuje same-field konflikt
Local F-1 status=Reviewing + remote F-1 Feature Status=Implementing → `sync conflicts`
neukázal F-1, ale ukázal false-positive na S-2 (iteration field, který failuje na push).
Diff mode F-1 odhalil jako field difference. Conflict logic potřebuje srovnání s last-pull
timestamp (changelog).

#### F12 — chybí enum validation pro status hodnoty
Test plán §7.1 vede uživatele nastavit Feature `status: Reviewing`, ale Feature lifecycle
v setup option_ids je {Funnel,Analyzing,Backlog,Implementing,Validating,Deploying,Releasing,Done}.
Žádný validátor neodmítne neplatnou hodnotu. Push pak selže rumtime. Návrh:
`validate_syntax.py` přečte `edpa.yaml.sync.option_ids` a enforce enum per type.

### MEDIUM — kosmetické, dokumentační

#### F1 — `issue_types.py` GraphQL update bug
```
GraphQL error: gh: Argument 'issueTypeId' on InputObject 'UpdateIssueTypeInput' is required.
InputObject 'UpdateIssueTypeInput' doesn't accept argument 'id'
```
Update Feature description selhal, ale create-or-skip větev funguje. Aktualizovat na
nový vstupní tvar `issueTypeId` místo `id`.

#### F5 — `validate_syntax.py` /dev/stdin no-op
`echo "id: BAD: bad" | validate_syntax.py /dev/stdin` říká "All files valid." (exit 0)
i pro syntakticky špatný YAML. /dev/stdin se evidentně nečte korektně.

#### F6 — `validate_syntax.py` neověřuje schema (jen YAML syntax)
Soubor s `id: S-2 / type: Story` (chybí title, parent, js, status, iteration) projde.
Test plán §3.2 očekává schema validaci.

#### F7 — `detect_contributors.py` chybí CLI mode
Test plán: `detect_contributors.py --item S-1 --since 7days`. Skript vyžaduje
`PR_NUMBER` env var (CI mode), CLI flags nemá.

#### F9 — `transitions.py --since` chybí relative format
`--since 1day` selže `ValueError: Invalid isoformat string: '1day'`. Dokumentován
formát `YYYY-MM-DD` (test plán je out-of-sync). Doporučení: přidat parser pro `Nday|Nweek|Nmonth`.

#### F11 — Feature Status options chybí "Reviewing"
Detail výše (Finding F12). Pokud je to záměr, test plán §7.1 patří opravit.

#### F18 — `evaluate_cw.py --auto-calibrate` neexistuje
Skript přijímá jen `--ground-truth` a `--heuristics`. Auto-calib loop je realizován
LLM skillem `/edpa:autocalib`, který sám rozhoduje o gating. Test plán §11.1 patří
přepsat na výzvu skillu (nebo přidat shell wrapper).

### LOW

- **Iteration YAML format** — `transitions.py` očekává `iteration: { start_date, end_date }`,
  ale `project_setup.py --bootstrap` produkuje plochý formát PI-level. Bootstrap pro
  per-iteration soubory (`PI-2026-1.{1..4}.yaml`) chybí — uživatel je musí psát ručně.
- Setup vypisuje "Issue type → Initiative" ale verifikace cíle není vidět.
- Snapshot `frozen_at: None` (snapshot.json) — pole není naplněné při zápisu.
