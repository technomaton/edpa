## Override scenario verification

Iteration `PI-2026-1.5` (IP iterace) deklaruje 3 overrides v
`iteration.people[]`:

```yaml
people:
  - id: bob-dev
    capacity_per_iteration: 44
    note: "IP weekend deploy push (Jun 13-14)"
  - id: alice-arch
    capacity_per_iteration: 10
    note: "vacation Jun 9-11 (3 days PTO)"
  - id: carol-qa
    capacity_per_iteration: 14
    note: "flu Jun 10-12 (sick leave certified)"
```

Engine output:

```
EDPA 1.9.0-beta — Iteration PI-2026-1.5 (simple mode)
======================================================================
Person                    Role     Capacity  Derived  Items   OK
----------------------------------------------------------------------
Bob                       Dev         44.0h    44.0h      1   OK   ← +4h overtime
Alice                     Arch        10.0h    10.0h      1   OK   ← -10h PTO
Carol                     QA          14.0h    14.0h      1   OK   ← -16h sick
Dave                      PM            10h    10.0h      1   OK   ← baseline (no override)
----------------------------------------------------------------------
TEAM TOTAL                            78.0h    78.0h
PLANNING CAPACITY                     62.4h  (factor: 0.8)

All invariants passed: YES
```

Math check: `44 + 10 + 14 + 10 = 78h` ✓ (effective capacity sum =
team_total derived). Σ DerivedHours = capacity invariant zachován pro
každou osobu.

## Pilot impact

EDPA v1.9.0-beta odstraňuje workaround A/B z kashealth pilot RFC
§ 1 ("není problém ruční přetížení, pokud je kde?"). Teď je kde:
přímo v iteration YAML přes `people:` blok, který tým už zná z
people.yaml. `KASHEALTH-PILOT.md` § 14 (otevřené otázky) může být
doplněna o ukázku override use-case pro PI-2026-2.5 IP iteraci.
