## Klíčové důkazy oprav

### N2 (auto-commit setup state) — `docs/E2E-REPORT-2026-05-06-v180.md` finding

**Reprodukce v1.8.0-beta:** Po `project_setup` byly mutace `.edpa/config/edpa.yaml` (field_ids/option_ids) a nový `.edpa/config/issue_map.yaml` uncommitted. Při PR squash merge na main + `git pull --ff-only` se working tree resetoval na HEAD, který měl pre-setup verzi → state ztracen.

**v1.8.1-beta scenário:**

```
git log před setupem:
  b0dbe18 EDPA: install + seed configs + backlog
  52707ce init

setup RUN 1:
  [9b] Committing setup state to git
       ✓ Setup state committed (.edpa/config/* + .edpa/iterations/)

git log po setupu:
  af4bd12 EDPA: persist setup state for project #42   ← AUTO-COMMIT ✓
  b0dbe18 EDPA: install + seed configs + backlog

→ vytvořena unrelated feature/S-1-readme-add branch + PR #5
→ gh pr merge --squash (PR #5 squashed do main)
→ git pull --ff-only

git log po PR merge + pull:
  06f6b96 feat(S-1): add v1.8.1 section (#5)
  af4bd12 EDPA: persist setup state for project #42   ← STATE PŘEŽIL ✓
  b0dbe18 EDPA: install + seed configs + backlog
  52707ce init

State check:
  project: 42
  Status fields: [Epic, Feature, Initiative, Status, Story]   (5/5)
  issue_map: I-1=#1, E-1=#2, F-1=#3, S-1=#4   (4/4)
  iteration files: PI-2026-1.{1..5}.yaml + PI-2026-1.yaml   (6/6)
```

**v1.8.0-beta** by v tomto bodě měl prázdný `field_ids`. **v1.8.1-beta** má kompletní state v gitu.
