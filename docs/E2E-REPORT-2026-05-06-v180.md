## Nové nálezy v tomto běhu

### N1 — `sync.cmd_conflicts` referenced non-existent helper (post-1.7 release)
- **Severity:** medium (regression caught before reaching kashealth)
- **Reprodukce:** sync conflicts po sync push s remote-side úpravou typed Status field
- **Stack:** `NameError: name 'parse_remote_items' is not defined` v `cmd_conflicts`
- **Root cause:** F10 fix v `sync.py` použil `parse_remote_items()` jako pojmenování, ale skutečná helper je `map_gh_items_to_edpa(gh_data, fields_mapping)`
- **Fix:** commit `46f0e1c` (3 lines, replace call). Sync.py také zkopírován do sandboxu pro zbytek E2E
- **Status:** ✅ commitnuto + pushnuto

### N2 — Setup state mutace `.edpa/config/edpa.yaml` po `project_setup` zůstávají uncommitted
- **Severity:** low (UX, ne crash)
- **Reprodukce:** `project_setup.py` mutuje `edpa.yaml` (přidá field_ids/option_ids) + vytvoří `issue_map.yaml`. Tyto změny nejsou auto-committed. Při `git checkout` na jinou branch nebo squash merge cesty se uncommitted změny mohou ztratit (E2E narazil — sandbox potřeboval re-run setup po PR merge).
- **Návrh:** setup script by mohl na konci nabídnout / dělat `git add .edpa/config/edpa.yaml .edpa/config/issue_map.yaml` (nebo aspoň jasně varovat uživatele "Please commit setup state to avoid loss")
- **Status:** trackováno pro v1.8.1 nebo 1.9.0
