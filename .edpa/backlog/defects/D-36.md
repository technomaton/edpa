## Description

Re-tune the `yaml_edit_*` weight set so a backlog YAML edit no longer
out-credits a real code commit, now that `commit_author` is the
calibrated 4.00 (D-35). Old `yaml_edit:create` was 5.0 вЂ” *larger* than a
code commit at the stale 2.78 anchor, and still в‰Ґ the 4.00 anchor. The
re-tune pegs `create` to half a code commit (2.0) and scales the finer
deltas down proportionally. Also: drop the dead
`yaml_edit:contributors_rebalance` key (consumer removed in D-26;
`score_diff` never reads it), add the consumed-but-absent
`bulk_item_threshold: 5` to the live config so it matches the template,
and bound every `yaml_edit_weights` per-signal value to `[-2.0, 8.0]` in
`validate_syntax` so a typo (`create: 50`) is rejected.

Weight changes (applied in lockstep across `.edpa/config/cw_heuristics.yaml`,
`plugin/edpa/templates/cw_heuristics.yaml.tmpl`,
`plugin/edpa/scripts/yaml_edit_signals.py::DEFAULT_WEIGHTS`, and the
vendored `.edpa/engine/` mirrors):
