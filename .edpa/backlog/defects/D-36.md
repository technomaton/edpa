## Description

Re-tune the `yaml_edit_*` weight set so a backlog YAML edit no longer
out-credits a real code commit, now that `commit_author` is the
calibrated 4.00 (D-35). Old `yaml_edit:create` was 5.0 — *larger* than a
code commit at the stale 2.78 anchor, and still ≥ the 4.00 anchor. The
re-tune pegs `create` to half a code commit (2.0) and scales the finer
deltas down proportionally. Also: drop the dead
`yaml_edit:contributors_rebalance` key (consumer removed in D-26;
`score_diff` never reads it), add the consumed-but-absent
`bulk_item_threshold: 5` to the live config so it matches the template,
and bound every `yaml_edit_weights` per-signal value to `[-2.0, 8.0]` in
`validate_syntax` so a typo (`create: 50`) is rejected.

Weight changes (applied in lockstep across `.edpa/config/cw_heuristics.yaml`,
`plugin/edpa/templates/cw_heuristics.yaml.tmpl`,
`plugin/edpa/scripts/yaml_edit_signals.py::DEFAULT_WEIGHTS`, and the
vendored `.edpa/engine/` mirrors):

| key | old | new |
|---|---|---|
| `yaml_edit:create` | 5.0 | 2.0 |
| `yaml_edit:scalar_change` | 0.5 | 0.25 |
| `yaml_edit:block_add` | 2.0 | 1.0 |
| `yaml_edit:list_grow` | 1.0 | 0.5 |
| `lines_volume_cap` | 3.0 | 1.0 |
| `lines_volume_divisor` | 30 | 40 |
| `yaml_edit:revert` | −0.5 | −0.5 (unchanged) |
| `list_grow_cap_per_commit` | 10 | 10 (unchanged) |

This is **forward-only**: historical `evidence[]` is frozen and deduped
by `ref`, so no past iteration is rescored. New weights take effect only
on commits made after this change.

## Impact model

Read-only recompute (no `evidence[]` was modified). `cw[P] = score[P] /
Σ score[*]` per item.

### (i) S-60 pilot case

- `daniel-matousek` = 4× `commit_author` + yaml_edit signals
  `[8.93, 1.0, 2.0, 0.5]` (8.93 = create + 7× scalar + vol; re-priced
  structurally under the new weights → 8.93→4.08, others 1.0→0.5,
  2.0→1.0, 0.5→0.25; daniel yaml_edit total 12.43 → 5.83).
- `jaroslav-urbanek` = 1× `commit_author` + `manual:commit_message` 13.0
  (manual weight is verbatim, never re-weighted by this table).

| split | daniel score | jaroslav score | cw daniel | cw jaroslav |
|---|---|---|---|---|
| BEFORE (`commit_author`=2.78 + old ye) | 23.55 | 15.78 | **0.5988** | 0.4012 |
| AFTER (`commit_author`=4.00 + new ye) | 21.83 | 17.00 | **0.5622** | 0.4378 |

The yaml_edit-heavy contributor's share drops ~3.7 pts; the
manual-attribution + single-commit contributor's share rises
correspondingly. This is the bug the re-tune targets.
